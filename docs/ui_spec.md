## 1. 共通仕様（スマホ・PC共通）

### 1-1. 1ターンの操作フロー

```
[ターン開始] → タイマー60秒スタート 【v1.3更新】
     │
     ├── コマ選択 → 指示入力（移動先/ドリブル/パス先/シュート/交代）
     │      │
     │      └── 指示確定（コマに矢印表示）→ 次のコマへ
     │
     ├── 指示済みコマの取消 → 再入力
     │
     └── [ターン確定ボタン] or [タイマー0秒] → 全指示を送信
              │
              └── 「実行」表示（約2.5秒）→ 次ターンへ 【v1.3更新：リプレイ→実行】
```

### 1-2. コマの状態表示（共通ルール）

| 状態 | 表示 |
|---|---|
| 未選択・未指示 | 通常色＋軽い点滅（未操作の注意喚起） |
| 選択中 | 黄色の強調リング。移動可能HEXがハイライト |
| 指示済み | 指示方向の矢印を表示。コマ色がやや暗くなる（操作完了感） |
| ボール保持 | コマの下にボールアイコン |
| 交代予定 | 点線の枠に変化（このターン行動不可を示す） |

### 1-3. ボード上の情報レイヤー

| レイヤー | 表示内容 | 常時/条件 |
|---|---|---|
| ゾーン境界線 | サード・レーンの境界を薄い線で表示 | 常時（ON/OFF切替可） |
| ZOC表示 | 選択中コマの周囲6HEXを青、相手コマのZOCを赤で表示 | コマ選択時のみ |
| 移動可能範囲 | 移動力内のHEXを緑でハイライト。ZOC侵入HEXは赤緑の縞 | コマ選択時のみ |
| パスライン | パス先を指定すると、パスコースを線で表示。相手ZOC/ZOC2通過時は警告色 | パス入力時 |

| ロングパス警告 | 正確パス距離(6HEX+コスト3+OM)を超えるパス命令に「LONG (NHex)」警告 | パス命令時 | 【v1.4追加】

1. タップ先に味方コマ → パス命令

3. それ以外 → ドリブル命令

明示モード（D:ドリブル / Q:パス / W:シュート）を設定している場合はそのモードで処理。

### 2-7. ~~プリセット行動の詳細~~ 【v1.3更新：廃止】

~~1分で11枚の操作が間に合わないことを前提に、複数コマに一括で「方向」を指示する補助機能。~~
**v1.3で廃止。個別コマ操作のみで十分と判断。**

**「前方」の定義**：各プレイヤーの攻撃方向を「前方」とする。Player Aが画面下→上に攻める場合、前方＝Y座標減少方向。Player Bは逆。サーバー側で各プレイヤーの攻撃方向に応じて座標を反転処理する。

| プリセット | 動作 |
|---|---|
| 全体前進 | 指示なしの全コマを、それぞれ移動力の範囲で1HEX前方（攻撃方向）に移動 |
| 全体後退 | 同上、1HEX後方に移動 |
| 守備ブロック | DF・SB・VO・GKを自陣方向に1HEX移動。MF以上は静止 |
| 攻撃展開 | MF・OM・WG・FWを前方に1HEX移動。DF以下は静止 |

**ZOC警告**：プリセット適用時、移動先が相手ZOC圏内に入るコマは**自動除外**し、静止状態のまま残す。画面上にはZOC除外されたコマ数を「⚠ 3枚ZOC除外」と表示。プレイヤーは除外されたコマを個別操作で移動先を指定できる。

> プリセットはターン開始時に最初に適用し、その後個別コマの指示で上書きする運用。「まずプリセットで安全な8〜9枚を動かし、ZOC付近の2〜3枚を個別に精密操作」が基本プレイ。

## 4. ミニゲームUI

### 4-1. FK（フリーキック）UI

**スマホ：**
- ゴールを6分割した大きなゾーンボタンを画面上部に表示
- 「直接/ロブ」の切替トグルを中央に配置
- 守備側は壁の高さ（低い/高い）のトグル＋GKダイブ方向の6ゾーンボタン
- 5秒のカウントダウン後、同時に結果表示

**PC：**
- ゴール正面のアングルをボード上に拡大表示
- マウスでゾーンをクリック。ホバーで各ゾーンの予測が表示
- キーボード：テンキーで6ゾーンに対応（7=左上, 8=中央上, 9=右上, 4=左下, 5=中央下, 6=右下）

### 4-2. CK（コーナーキック）UI

**スマホ：**
- PA内を3ゾーン（ニア/中央/ファー）に大きく分割
- 攻撃側：まず投入3枚を選択（スワイプで選択）→各コマをゾーンにドラッグ配置
- 守備側：同様に3枚選択→ゾーンにドラッグ配置
- 10秒のカウントダウン

**PC：**
- PA拡大表示。3ゾーンをマウスクリックで選択
- 左パネルからコマをドラッグ＆ドロップでゾーンに配置
- ショートカット：1/2/3キーでゾーン選択

### 4-3. PK UI

**スマホ・PC共通：**
- ゴール正面の大きな画面。6ゾーンをタップ/クリック
- キッカーとGKの情報（コスト・ポジション）を画面上部に表示
- PK戦時は蹴り順リストと現在のスコアを表示

**PK戦のキッカー選択UI：**
- フィールド11枚＋ベンチ全員のコマ一覧を表示
- 各コマをドラッグ（スマホ）またはクリック（PC）で蹴り順スロット1〜5に配置
- コスト・ポジションが表示され、シュート修正のあるコマにはアイコン表示
- 選択完了で「確定」ボタン

## 4-4. 切断・再接続UI

| 状態 | 表示 |

## 5. 解決アニメーション

ターン確定後、両者の行動が解決される過程をアニメーションで表示する。

### 5-1. アニメーション順序（処理順序§9-2と整合）

| フェーズ | アニメーション | 時間 |
|---|---|---|
| 1. コマ移動 | 全コマが同時にスライド移動。ZOC停止コマは途中で止まる | 0.8秒 |
| 2. 競合・タックル | 同一HEX競合やタックルの結果をエフェクトで表示 | 0.3秒 |
| 3. ファウル判定 | ファウル成立時はホイッスル＋FK/PK遷移表示 | 0.3秒 |
| 4. ボール処理 | パスのボール軌跡 or シュートのボール軌跡を描画 | 0.5秒 |
| 5. パスカット/ブロック/セーブ | 成否をエフェクトで表示 | 0.3秒 |
| 6. オフサイド判定 | オフサイド成立時は旗＋ホイッスルで表示 | 0.3秒 |
| **合計** | | **約2.5秒** |

> アニメーション順序は仕様書§9-2の処理順序（移動→ボール→特殊判定）と一致させる。

## 6. 共通UIコンポーネント

### 6-1. コマアイコン

コマアイコンの設計優先順位：①敵味方の区別 ②ランク ③ポジション種別

| 要素 | 仕様 |
|---|---|
| 形状 | 円形（通常: 64px / SS: 72px） |
| 敵味方の区別 | 味方=青系(#2563EB)、敵=赤系(#DC2626) |
| 中央テキスト | ランクを大きく表示。白文字、font-weight: 500 |
| ランク表記 | 1, 1+, 2, 2+, SS の5種類（コスト1→1, 1.5→1+, 2→2, 2.5→2+, 3→SS） |
| ポジション表示 | 右上に小さなバッジ（略称2文字） |

**ランク別枠装飾：**

| ランク | 表記 | 枠 | サイズ |
|---|---|---|---|
| コスト1 | 1 | 枠なし（2px同色ボーダー） | 64px |
| コスト1.5 | 1+ | 銅枠（3px, #CD7F32） | 64px |
| コスト2 | 2 | 銀枠（3px, #C0C0C0） | 64px |
| コスト2.5 | 2+ | 金枠（3px, #FFD700） | 64px |
| コスト3 | SS | 金枠（4px, #FFD700）＋外側リング（2px, #FFA500）＋サイズ大 | 72px |

SSのコマは背景にグラデーション、テキストは金色(#FFD700)。

**特殊状態：**

| 状態 | 表示 |
|---|---|
| 未選択・未指示 | 通常表示（軽い点滅で未操作を注意喚起） |
| 選択中 | 黄色枠(#FACC15)に切り替わり、枠が点滅アニメーション（1.2秒周期のpulse） |
| 指示済み | コマが半透明(opacity: 0.5)＋下部に「指示済」バッジ（緑） |
| ボール保持 | コマ下部にサッカーボールアイコン（28px、白地に五角形パターンのSVG） |
| 交代予定 | 点線の枠に変化（そのターン行動不可を示す） |

**ポジションバッジの色：**
- 味方：背景#1D4ED8、文字#fff
- 敵：背景#B91C1C、文字#fff
- SS：背景#92400E、文字#FDE68A
