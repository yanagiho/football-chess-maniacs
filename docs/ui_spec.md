## 1. 共通仕様（スマホ・PC共通）

### 1-1. 1ターンの操作フロー

```
[ターン開始] → タイマー60秒スタート
     │
     ├── コマ選択 → 指示入力（移動先/ドリブル/パス先/シュート/交代）
     │      │
     │      └── 指示確定（コマに矢印表示）→ 次のコマへ
     │
     ├── 指示済みコマの取消 → 再入力
     │
     └── [ターン確定ボタン] or [タイマー0秒] → 全指示を送信
              │
              └── 「実行」表示（約2.8秒）→ 次ターンへ 【v2.0更新：2.5秒→2.8秒】
```

### 1-2. コマの状態表示（共通ルール）

| 状態 | 表示 |
|---|---|

| 選択中 | 黄色の強調リング(#FACC15)。移動可能HEXがハイライト |
| 指示済み | **命令種別バッジ**を表示（左下に小さい丸）【v2.0更新】 |
| ボール保持 | コマの右下にボールアイコン（SVGサッカーボール） |
| 交代予定 | 点線の枠に変化（このターン行動不可を示す） |

**命令済みバッジ** 【v2.0追加】:

| 命令 | バッジ色 | アイコン |
|---|---|---|
| 移動 | 緑(#44aa44) | → 矢印 |
| パス | 青(#4488cc) | ⚽ |

## 2. スマホUI

### 2-1〜2-3.

（v1.4 と同一）

**ボール操作: コマ/ボール分離タッチ** 【v2.1更新】：

ボール保持者は「コマ本体」と「ボールアイコン」を別々にタッチできる。

■ **コマ本体をタッチ** → ドリブルモード:
- ガイドテキスト:「ドリブル先をタップ」
- 移動可能範囲（3HEX）を緑ハイライト
- HEXタップ → ドリブル命令確定（矢印: 緑）

■ **ボールアイコンをタッチ** → パス/スルーパス/シュートモード:
- ガイドテキスト:「パス先をタップ（味方=パス / 空きHEX=スルーパス / ゴール付近=シュート）」
- タップ先の自動判定:
  1. 味方コマ → パス命令（矢印: 青、ZOC通過でオレンジ）
  2. 空きHEX → スルーパス命令（矢印: シアン点線）
  3. シュートゾーン → シュート命令（矢印: 赤）

■ ボールアイコン: コマの右上に配置、コマの50%サイズ（32px）、白い丸+五角形パターン+drop-shadow

■ **チェーンパス** 【v2.1追加】:
- ボールタッチ後、味方へパス → その味方にボールが移動
- 再度ボールタッチで次のパスを出せる（何度でも繰り返し可能）
- 「キープ」ボタンでチェーン終了
- シュート or スルーパスでもチェーン終了
- パス回数を「Pass ×N」とカウント表示
- パス履歴の軌跡線を画面上に表示

■ **ボール非保持者** → 移動モード（従来通り）

明示モード（D:ドリブル / Q:パス / W:シュート）も引き続き使用可能。

### 2-4. アクションバー（下部固定）

| ボタン | 動作 | 条件 |
|---|---|---|
| ← 戻る | 直前の指示を取消 | 指示済みコマがある時 |
| ドリブル | ドリブルモードに切替 | ボール保持コマ選択時のみ |
| パス | パスモードに切替 | ボール保持コマ選択時のみ |
| シュート | シュートモードに切替 | ボール保持コマ選択時のみ |
| 交代 | ベンチ一覧がスライドアップ | 交代枠が残っている時 |
| ✓ 確定 | 全指示を送信してターン終了 | 常時 |

## 4. ミニゲームUI

### 4-1. FK（フリーキック）UI

- ゴール6分割ゾーンボタン（左上/中上/右上/左下/中下/右下）
- 「直接/ロブ」切替トグル
- GK側: 壁の高さ（低い/高い）+ ダイブ方向6ゾーン
- 5秒カウントダウン、0秒で自動送信
- **操作説明テキスト常時表示**: 「ゴールの狙う方向を選んでください」 【v2.0追加】
- PCキーボード: テンキー 7=左上, 8=中上, 9=右上, 4=左下, 5=中下, 6=右下

### 4-2. CK（コーナーキック）UI

- PA内3ゾーン（ニア/中央/ファー）
- 攻撃/守備: まず投入3枚選択 → 各コマをゾーンに配置
- 10秒カウントダウン
- **操作説明テキスト常時表示**: 「まず投入する3枚のコマを選び、次にゾーンに配置してください」 【v2.0追加】

### 4-3. PK UI

- ゴール6ゾーン選択（PKGame.tsx）
- 5秒カウントダウン
- PK戦時はPKShootout.txを使用（スコアボード+5本交互+サドンデス）
- **操作説明テキスト常時表示**: 「ゴールの蹴る方向を選んでください（6ゾーン）」 【v2.0追加】

### 4-4. 切断・再接続UI

5パターンの接続状態に対応（ConnectionBanner.tsx）:

| 状態 | 表示 | 色 |
|---|---|---|
| CONNECTED | なし（通常状態） | — |
| RECONNECTING | 上部バナー「接続が不安定です。再接続中...」+ スピナー | 黄(#cc8800) |
| OPPONENT_DISCONNECTED | モーダル + 30秒カウントダウン | — |
| DISCONNECTED | モーダル「接続が切れました」+ 「再接続する」ボタン | 赤 |
| MATCH_ABANDONED | モーダル「試合が終了しました」+ 理由 + 「ホームに戻る」 | — |

COM対戦時は非表示。自動再接続: 3秒間隔×最大5回リトライ。

## 5. 解決アニメーション

### 5-1. アニメーション順序 【v2.0更新】

| フェーズ | アニメーション | 時間 |
|---|---|---|
| 1. コマ移動 | 全コマが同時にスライド移動。ZOC停止コマは途中で止まる | **0.8秒** |
| 2. 競合・タックル | 同一HEX競合やタックルの結果をエフェクトで表示 | **0.5秒** 【v2.0更新：0.3→0.5秒】 |
| 3. ファウル判定 | ファウル成立時はホイッスル＋FK/PK遷移表示 | **0.5秒** 【v2.0更新：0.3→0.5秒】 |
| 4. ボール処理 | パスのボール軌跡 or シュートのボール軌跡を描画 | **0.5秒** |
| 5. パスカット/ブロック/セーブ/オフサイド | 成否をエフェクトで表示 | **0.5秒** 【v2.0更新：0.3→0.5秒】 |
| **合計** | | **約2.8秒** 【v2.0更新：2.5→2.8秒】 |

### 5-1b. エフェクト詳細

| イベント | エフェクト | 表示位置 |
|---------|-----------|---------|
| タックル成功 | ⚔ + 「TACKLE」テキスト | 発生HEX上 |
| タックル失敗 | 💨 + 「MISS」テキスト | 発生HEX上 |
| ファウル | 🟨 + 「FOUL」テキスト | 発生HEX上 |
| パス成功 | ⚽ ボール軌跡 | 始点→終点 |
| パスカット | ✋ + 「INTERCEPTED」テキスト | カットしたコマ上 |
| シュート（ゴール） | ⚽ 軌跡 + 「GOAL!」金色大テキスト | ゴール方向→画面中央 |
| シュート（ブロック） | 🛡 + 「BLOCKED」テキスト | ブロックしたコマ上 |
| シュート（セーブ） | 🧤 + 「SAVED」テキスト | GK位置 |
| シュート（キャッチ） | 🧤 + 「CATCH」テキスト | GK位置 |
| シュート（外れ） | 💨 + 「WIDE」テキスト | ゴール横 |
| 競合勝ち | 💪 | 勝者コマ上 |
| 競合負け | 💫 | 敗者コマ上 |
| オフサイド | 🚩 + 「OFFSIDE」テキスト | オフサイド位置のコマ上 |

**描画仕様** 【v2.0更新】:
- アイコンフォント: **bold 40px sans-serif**（v1.4の18pxから拡大）
- テキストフォント: **bold 24px sans-serif**（v1.4の10pxから拡大）
- Canvas上にフェーズ時間内でフェードイン→フェードアウト
- 同時に複数イベント発生時は全て同時表示

### 5-2. スキップ機能

**廃止**。「実行」表示（約2.8秒）後に自動遷移。8秒安全タイムアウトあり。

実装: PHASE_TIMINGS = [800, 500, 500, 500, 500] (ms)。Piece.tsxのCSS transition (0.8s ease-out)でコマ移動。フェーズ1-4はCanvasオーバーレイにエフェクト描画。
