## 0. このドキュメントの位置づけ

- 対象: アウトゲーム（マイページ〜対戦開始まで）の再設計
- ゴール: **「自分のチームがどれか」「誰と戦うか」を常に一目で分かる状態にする**
- 前提: 必要な素材（NPC_TEAMS のエンブレム/名前/Era、編成、ショップ）はすでに実装済み。**新規実装ではなく「配線の組み替えと表示の追加」が中心**
- 既存仕様を削ってよい: `TeamSelect` 相当の再導入はしない。旧「クイックマッチ／モード選択／プリセットチーム」の3入口分立は本書で解消する

## 1. 設計意図（最重要）

### 1.1 「自チーム」を一級市民にする

現状の最大の問題は、**プレイヤーが今どのチームを使っているのか、どこにも表示されていない**こと。
`Formation.tsx` はコマの寄せ集めを組む画面でしかなく、チーム名もエンブレムもない。
一方 `src/data/npc_teams.ts` にはすでに名前・絵文字・Era・フォーメーション名が揃っている（`PresetTeamsScreen.tsx` でしか使われていない）。

→ **マイページ**を新設し、「今の自チーム」を常時1枚のカードとして可視化する。ショップ（選手購入）・編成（スタメン+配置）はすべてこのマイページから出入りする寄り道として位置づける。

### 1.2 「対戦相手」も一級市民にする

COM対戦の相手は `createDefaultAwayPieces()` で常に同じ無名の4-4-2固定チームが生成されており、対戦相手の名前もエンブレムも一切表示されない。

→ COM対戦時は `NPC_TEAMS` から相手を選出し、対戦開始前に「対戦相手：⚡️〇〇（Era3）」を必ず見せる。

### 1.3 対戦は3種類に整理する

| 種別 | 内容 |
|---|---|
| COM対戦 | 難易度選択 + 対戦相手プレビュー（NPC_TEAMSから選出） |
| オンライン対戦 | ランク/カジュアル。マッチング人数が不足する場合はBotで補完 |
| フレンド対戦 | 招待コード／URLを発行し、コピペで相手に共有。相手はコード/URL入力で参加 |

COM観戦（COM同士の自動対戦）は上記3種とは別枠の小さい機能として残す（動作確認用途）。

## 2. 現状の動線（整理前）

```
タイトル
 ├─「前回の編成で対戦」──────▶ マッチング
 ├─「モードを選んで対戦」──▶ モード選択（ranked/casual/com/comVsCom + 難易度）
 │                              ├─「編成して開始」──▶ 編成
 │                              └─「この設定で開始」──▶ マッチング
 ├─「編成をいじる」────────▶ 編成
 ├─「フレンド対戦」────────▶ FriendMatchScreen（モック・API未接続）
 └─「プリセットチーム」────▶ プリセット選択 → 即COM対戦開始（モード選択を経由しない隠れ導線）
```

### 洗練を妨げている主因

1. **試合を始める入口が3つに分立**（クイックマッチ／モード選択／プリセットチーム）。相互の関係が画面上どこにも説明されない
2. **「自分のチーム」に見た目のアイデンティティがない**。チーム名・エンブレムがどこにも出ない
3. **COM対戦の相手が常にブラックボックス**。`createDefaultAwayPieces()` が常に同じ無名チームを生成
4. **フレンド対戦がモックのまま公開導線に出ている**
5. タイトルのボタンが10個（メイン4＋サブ6）あり、全体像が一望しづらい

## 3. 整理後の動線（正本）

```
[マイページ（自チームハブ）]
   自チームカード（エンブレム・チーム名・スタメン概要）
   ├─「ショップへ」──▶ [ショップ]（選手購入）─┐
   ├─「編成する」────▶ [編成]（スタメン+配置、1画面完結）─┤
   │                                                    └─▶ マイページへ戻る（カード更新）
   └─「対戦へ」──▶ [対戦タイプ選択]
                     ├─ COM対戦 ──▶ 難易度選択+相手プレビュー ──▶ [マッチング]
                     ├─ オンライン対戦 ──▶ ランク/カジュアル ──▶ [マッチング]（Bot補完あり）
                     ├─ フレンド対戦 ──▶ 招待コード/URL発行 or 参加 ──▶ [マッチング]
                     └─ COM観戦（小さく配置） ──▶ [マッチング]

[マッチング] （自チーム vs 対戦相手を名前・エンブレム表示）──▶ [対戦] ──▶ [リザルト]

[リザルト] ─┬─「もう一度」────▶ [マッチング]
            ├─「編成を変える」─▶ [編成]
            └─「マイページへ」──▶ [マイページ]

[補助機能]（マイページから随時アクセス、試合フローとは別軸）
   コレクション / ランキング / プロフィール / リプレイ / 設定
```

### 変更点サマリ

| 変更 | 内容 | 効果 |
|---|---|---|
| マイページ新設 | `Title.tsx` を「自チームカード」中心のハブに再設計 | 自分のチームが常に見える |
| チーム識別情報の追加 | 自作編成にも名前/代表アイコンを付与（プリセットは既存の name/emoji を流用） | 「今このチームを使っている」が一目瞭然 |
| 対戦相手の可視化 | COM戦は `NPC_TEAMS` からランダム選出し、開始前にプレビュー表示 | 「誰と戦うか」が試合前に分かる |
| 対戦タイプの3分類 | COM対戦／オンライン対戦／フレンド対戦を並列の選択肢に統合 | 入口の分立を解消 |
| オンラインへのBot補完 | マッチング人数不足時にBotを割り当てる | 待ち時間短縮 |
| フレンド対戦の実装 | 招待コード/URL発行→コピペ共有、参加側はコード/URL入力 | モックから実機能へ |
| プリセットチーム画面の位置づけ変更 | 独立導線を廃止し「編成」内の選択肢（自作 or プリセットから選ぶ）に統合 | 入口の重複解消 |
| COM観戦の格下げ | 対戦タイプ選択内に小さく残す（動作確認用） | 表に出しすぎない |

## 4. 実装計画（Claude Code 作業単位）

> 既存はフルスペック。多くは「既存画面の再配置・表示追加」。対象ファイルは現状把握用の目安。

### Phase 1: 自チームの可視化基盤（最優先）

**T1. チーム識別情報の型定義**
- `LastSetup`（`src/client/utils/lastSetup.ts`）に `teamName` / `teamEmoji` / `origin`（'custom' | 'preset'）を追加
- 自作編成時は `formation.teamName` を保存できるようにし、未設定時は「マイチーム」をデフォルト表示
- プリセット選択時は `PresetTeam.name` / `emoji` をそのまま引き継ぐ
- 受け入れ条件: 編成・プリセットどちらで確定しても、名前とエンブレムが確定する

**T2. マイページ画面の新設（`Title.tsx` を置き換え）**
- 起動直後に「自チームカード」（エンブレム・チーム名・スタメン人数/コスト合計）を最上段に常時表示
- カードから「ショップへ」「編成する」「対戦へ」の3導線
- 受け入れ条件: 起動して最初に見える画面で「今のチーム」が一目で分かる

**T3. COM対戦相手の可視化**
- `battleUtils.ts` の `createDefaultAwayPieces()` を廃止し、`NPC_TEAMS` からランダム（または難易度連動）で1チーム選出する関数に置き換え
- 選出結果を対戦タイプ選択画面・マッチング画面に「対戦相手：{emoji}{name}（Era{n}）」として表示
- 受け入れ条件: COM対戦開始前に必ず相手のチーム名とエンブレムが見える

### Phase 2: 対戦タイプの再編

**T4. 対戦タイプ選択画面（`ModeSelect.tsx` 改修）**
- 「COM対戦／オンライン対戦／フレンド対戦」を並列カードで提示し、COM観戦は下部に小さく配置
- COM対戦選択時のみ難易度＋相手プレビュー（T3）を展開
- オンライン対戦選択時のみランク/カジュアルを展開
- 受け入れ条件: 3種の対戦タイプが同一画面で選べ、選択に応じて必要な設定だけ表示される

**T5. オンライン対戦のBot補完**
- マッチメイキングDO（`durable/matchmaking.ts`）に、一定時間人が見つからない場合Botを割り当てるフォールバックを追加
- 受け入れ条件: オンライン対戦で対戦相手が見つからない場合も一定時間内に試合が開始する

**T6. フレンド対戦の実装（`FriendMatchScreen.tsx` のモック解消）**
- 招待側: 招待コード（短い英数字）とマッチURLを発行するAPIを新設し、コピペ用に画面表示
- 参加側: コードまたはURL（クエリパラメータ経由）で該当セッションに合流
- 受け入れ条件: 招待コード/URLをコピペで共有し、相手がそれだけで合流できる

### Phase 3: 整理・削減

**T7. 旧3入口の統合**
- タイトルの「クイックマッチ／モード選択／プリセットチーム」ボタンを廃止し、マイページの「対戦へ」1本に集約
- 受け入れ条件: 対戦を始める入口がマイページの「対戦へ」のみになる

**T8. プリセットチーム画面の位置づけ変更**
- `PresetTeamsScreen.tsx` への独立導線を廃止し、「編成」画面内の選択肢（自作編成 / プリセットから選ぶ）として統合
- 受け入れ条件: プリセットチームは「編成する」経由でのみアクセスでき、独立ボタンが消える

---

### Phase 4: ビジュアルの主役化（参考: grassrootsfootball.football との比較）

**T9. マイページ・対戦タイプ選択の視覚的リメイク**
- 背景差分（2026-07-01）: `https://grassrootsfootball.football/` との比較で、情報設計は近いが「視覚的な主役の作り方」に差があると判明。具体的に4点：
  1. 自分のアイデンティティ（アバター/エンブレム）が画面中央に大きく主役として置かれている
  2. メイン導線が「PLAY NOW」「ONLINE MATCH」の巨大2択ボタンに絞られている
  3. サブ機能（Customize/Room List/Club/Shop/Profile/Settings）が3列×2行の整ったグリッドで畳まれている
  4. 黒×黄色の強いコントラストで「ゲームを開いた」感が一発で伝わる
- 対応方針:
  - **T9a. 自チームカードの格上げ**: `MyPage`（旧Title.tsx）の自チームカードを、エンブレム絵文字＋テキストの小さいサブラベルから、画面中央の大きめビジュアルカード（エンブレム/代表アイコンを主役サイズで表示）に変更
  - **T9b. 対戦導線の2択化**: T4で作った「COM対戦／オンライン対戦／フレンド対戦」の3枚並列カードを、「今すぐCOM対戦」「オンライン対戦」の2大ボタンに再編し、フレンド対戦はその下に控えめな3番目の選択肢として配置（COM観戦はさらに小さく最下部）
  - **T9c. 補助機能のグリッド化**: コレクション/ランキング/プロフィール/リプレイ/設定/ショップの6つを、横並びpillの`flex-wrap`から3列×2行の均等グリッドに変更
  - **T9d. 配色コントラスト強化**: アクセントカラー（現状の`#ffd700`系グラデーション）の主張を強め、ベース配色とのコントラストを上げる
- 受け入れ条件: マイページ起動直後に「自分のチーム」が画面の主役として視認でき、対戦への主導線が2つの巨大ボタンに絞られ、補助機能が整ったグリッドで表示される
