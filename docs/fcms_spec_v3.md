## 2. ゲームコンセプト

### キャッチコピー
> **「あの歴史の瞬間に、あなたがいた。」**
> **「将棋・チェスのようにサッカーを楽しむ、戦術対戦ゲーム 第2弾」**

### 概要
Football Chess ManiacS（以下 ManiacS）は、前作「Football Chess」の大幅パワーアップ版。プレイヤーは監督として選手コマを配置し、ターン制でコマを動かしてサッカーの戦術を再現する対戦ゲームである。

コマとなる選手は、**GrassRoots世界観**の中に生きるオリジナルキャラクター。1863年〜2020年代のサッカー史の中から各時代のスター選手が登場し、時代・地域・境遇を超えたドリームチームを編成して戦う。

### 4-4. コマのコストシステム

すべてのコマは **コスト（1〜3）** を持つ。コストはコマの総合力を表し、ゲーム内のあらゆる判定式の基幹変数となる。

| コスト | 意味 | 移動・パスへの影響 |
|---|---|---|
| 1 | 一般的な選手 | 基本値のまま |
| 1.5 | やや優秀 | 基本値のまま（判定式で特殊処理） |
| 2 | 優秀な選手 | 基本値のまま |
| 2.5 | かなり優秀 | 基本値のまま（判定式で特殊処理） |
| 3 | スター選手 | 移動+1、正確パス射程+1 |

> **1.5 / 2.5の特殊処理**：コスト差の計算で、片方が.5で相方が整数の場合は小数のまま差を取る。両方が.5の場合は通常の整数処理。

## 6. ZOC（Zone of Control）仕様

### 6-1. ZOC定義

| 名称 | 範囲 | 用途 |
|---|---|---|
| **ZOC** | コマの隣接6HEX | 移動停止、シュートブロック、タックル、パスカット2 |
| **ZOC2** | ZOCの外周12HEX | パスカット1 |

### 6-2. ZOCによる移動停止

相手コマのZOC内に進入した時点で**移動が強制停止**される。

| 状況 | 処理 |
|---|---|
| ボール非保持コマが相手ZOCに進入 | そのHEXで停止。残り移動力は消失 |
| ドリブルコマが相手ZOCに進入 | そのHEXで停止。タックル判定（§7-4）が発生 |
| 移動先が相手ZOCでない場合 | 通常通り移動力を消費して進行 |

> ZOCは「通過」できない。迂回するか、相手コマを排除してZOCを消すか、スキルでZOCを無効化する必要がある。

### 6-3. ZOCのその他の効果
- パスコースが相手ZOC/ZOC2を通過する場合、パスカット判定が発生する
- シュートコースが相手ZOC内を通過する場合、シュートブロック判定が発生する
- 各判定式において、ZOC内の味方/敵コマの数が確率修正に影響する

## 7. 判定システム

### 7-1. 基本判定式

```
成功確率（%）＝（x − y + 3）× Ω ＋ ポジション修正 ＋ ZOC隣接修正
```

- **x** = 判定を仕掛ける側のコマのコスト
- **y** = 判定を受ける側のコマのコスト
- **Ω** = アクションごとの基本係数
- コスト差の範囲は−2〜+2、基本値は **1Ω〜5Ω** の5段階

### 7-2. シュート判定チェーン

**① シュート入力** — コースを選択。相手コマの影響がなければ100%成功。シュートコマのZOC内に守備側コマがいる場合、1体につき−15%。

**② シュートブロックチェック** — コースが守備側コマのZOC内を通る場合に発生。

| 要素 | 値 |
|---|---|
| 基本式 | Ω=10（守備側成功確率） |
| DF修正 | +15 |
| SB修正 | +10 |
| FW修正（シューター） | −10 |
| WG修正（シューター） | −5 |
| OM修正（シューター） | −5 |
| 攻撃側ZOC隣接 | 1体につき−5 |
| 守備側ZOC隣接 | 1体につき+10 |

ブロック成功時：**ブロックコマがボールを獲得**して次ターンへ。

**③ セービングチェック** — コースがGKのZOC内を通る場合に発生。

| 要素 | 値 |
|---|---|
| 基本式 | Ω=15（守備側成功確率） |
| FW修正（シューター） | −15 |
| WG修正（シューター） | −10 |
| OM修正（シューター） | −10 |
| 攻撃側ZOC隣接 | 1体につき−5 |
| 守備側ZOC隣接 | 1体につき+10 |
| 距離修正 | （GKまでのHEX数−2）×5 |
| GKのZOC内の守備コマ | 1体につき−10 |

セービング成功時：キャッチ判定へ。失敗時：シュート成功チェックへ。

**③-b キャッチ判定**

| 要素 | 値 |
|---|---|
| 計算式 | GKコマのコスト × 30 |

成功→GKがキャッチして次ターンへ。失敗→CKミニゲームへ。

**④ シュート成功チェック**

| 要素 | 値 |
|---|---|
| 計算式 | シュートコマのコスト × 5 + 70 |
| 攻撃側ZOC隣接 | 1体につき+5 |
| 守備側ZOC隣接 | 1体につき−10 |
| 距離修正 | （ゴールまでのHEX数−3）×−5 |

### 7-3. パスカット判定

**パスカット1（パスコース遮断）** — パスコースが守備側コマのZOCまたはZOC2を通る場合に発生。

| 要素 | 値 |

| 攻撃側ZOC隣接 | 1体につき+5 |
| 守備側ZOC隣接 | 1体につき−10 |

### 7-5. ファウル判定

タックル発生後にファウル判定を行う。**アタッキングサード（ゴールラインから12HEX以内）でのみ発生**。それ以外のゾーンではファウルにならない。

| 項目 | 内容 |
|---|---|
| 基本ファウル確率 | 25%（※要調整） |
| PA内特殊ルール | PA内に守備側コマが多数いる場合、必ずファウル（閾値は要確定） |
| PA内ファウル | → PKミニゲーム |
| PA外ファウル | → FKミニゲーム |

### 7-6. 同一HEX競合判定

両者のコマが同じHEXに移動しようとした場合に発生。

| 要素 | 値 |
|---|---|
| 基本式 | (x−y+3)×Ω |
| Ω | 15 |
| 勝者 | そのHEXに留まる |
| 敗者 | 移動前のHEXに戻される |
| ボール保持コマが関与 | タックル判定（§7-4）に切り替え |

## 9. ターン進行

### 9-1. ターン構造

| 項目 | 内容 |
|---|---|
| 入力方式 | **両プレイヤー同時入力** |
| 操作対象 | **フィールド上の全11枚に指示可能**（ただし制限時間内に全枚操作しきれないことを前提とする） |
| 制限時間 | 1ターンあたり1分。意図的にタイトな設定 |
| 総ターン数 | 90ターン（前半45・後半45） |
| 先攻後攻 | コイントス（ランダム）で決定 |

> **1ターンの流れ**：①両者が1分以内に全コマへの指示を同時入力 → ②入力完了または制限時間到達で締切 → ③同時解決
>
> **時間制限の設計意図**：1分で11枚すべてに指示を出しきれないことを前提とした設計。「どのコマを優先して動かすか」の判断自体がゲームの核心的スキルとなる。指示を出せなかったコマは現在位置に静止する。操作速度と戦況判断の両方が求められるため、上級者ほど少ない時間で的確にコマを動かせる＝実力差が出る構造。

### 9-2. 同時解決の処理順序

1ターンの全アクションは以下の4フェーズで解決される。

**フェーズ0：スナップショット** — ターン開始時の全コマ位置を記録する（オフサイド判定用）。

**フェーズ1：コマ移動** — 全コマが同時に移動する。
1. 移動指示のある全コマが同時に移動先へ向かう
2. 相手ZOCに進入したコマはその地点で停止
3. 同一HEX競合が発生した場合、競合判定（§7-6）
4. ドリブルコマが相手ZOCに進入した場合、タックル判定（§7-4）
5. タックルがアタッキングサードで発生した場合、ファウル判定（§7-5）。**ファウル成立時はタックル結果を無効化し、攻撃側にFK/PKを付与**

**フェーズ2：ボール処理** — フェーズ1の移動完了後の盤面でパス・シュートを解決する。
6. シュート判定チェーン（§7-2）。GKの移動後の位置でセービング判定
7. パスの配送：パス先のコマの**移動後の位置**にボールが届く（スルーパスが成立する）
8. パスカット1判定：パスコース上の守備コマの**移動後の位置**で判定
9. パスカット2判定：受け手コマの**移動後のZOC**内に守備コマがいるか判定

**フェーズ3：特殊判定**
10. オフサイド判定：**フェーズ0のスナップショット（移動前の位置）**を基準に判定。オフサイドライン＝移動前の守備側後方2番目のコマ。受け手の位置＝移動前の受け手の位置

> **スルーパスの仕組み**：FWに「前方移動」、OMに「FWへパス」を同時指示した場合、フェーズ1でFWが前方に走り、フェーズ2でOMのパスがFWの移動後の位置に届く。ただしオフサイドはフェーズ0の移動前位置で判定されるため、FWが移動前にオフサイドポジションにいればアウト。「オンサイドの位置から走り込んで受ける」という実際のサッカーと同じ駆け引きが再現される。
>
> **ファウルの優先順位**：タックルが成功し、かつファウルも成立した場合は**ファウルが優先**。守備側のボール獲得は取り消され、攻撃側にFK/PKが与えられる（サッカー準拠）。

### 9-3. 各コマへの指示

各コマには1ターンにつき**1つの指示**を出せる。指示を出さなかったコマは現在位置に留まる。

| 指示 | 対象 | 説明 |
|---|---|---|
| 移動 | 全コマ | ボール無しコマをHEXマス上で移動 |
| ドリブル | ボール保持コマ | ボールと共に移動。相手ZOC進入でタックル発生 |
| パス | ボール保持コマ | ボールを指定コマへ送る |
| シュート | ボール保持コマ | ゴールへボールを蹴る |
| 交代 | 全コマ | コマを入れ替える（§9-4参照） |
| スキル発動 | 全コマ | 時代スキルを使用（※後回し） |

#### ポジション固有指示

| 指示 | 対象 | 説明 |
|---|---|---|
| オーバーラップ | SB | サイドを縦に駆け上がる複合移動 |

### 9-4. 交代

| 項目 | 内容 |
|---|---|
| 操作 | 交代対象コマに「交代」指示を出し、ベンチから入れ替えるコマを選択 |
| 位置 | 交代対象コマの**現在HEXにそのまま出現** |
| 行動制約 | 交代で出場したコマは**そのターン行動不可**（静止状態） |
| コスト制約 | 交代後のフィールド総コスト16を超えてはならない |
| 回数 | 1試合5回まで |

> **設計意図**：交代コマがそのターン動けないため、交代した瞬間はそのHEXが「穴」になる。守備中に交代すると相手に突かれるリスクがある。攻撃の切れ目やハーフタイム直後（初期フォーメーションに戻るタイミング）が交代の好機。

### 9-5. オフサイドルール

**オフサイドライン**：守備側の後方から2番目のコマ（GK含む）が位置するHEX列。

**判定基準**：パスが出された瞬間の、**移動前の位置**で判定する（サッカー準拠）。守備側のラインも移動前の位置が基準。

| 受け手の位置（移動前） | 判定 |
|---|---|
| ラインより2HEX以上敵陣寄り | **確定オフサイド** |
| ラインより1HEX敵陣寄り | **グレーゾーン**（50%でオフサイド ※要調整） |
| ライン同列または自陣寄り | **オンサイド** |

オフサイド成立時：その地点から相手チームのボールで再開。

### 9-6. ボールの状態

| 状態 | 説明 |
|---|---|
| 保持 | コマがボールを持っている。そのコマのHEXにボールが存在 |
| ボール獲得 | タックル成功・パスカット成功・シュートブロック成功時、成功したコマがボールを獲得 |
| ゴールキック | ボールが相手ゴールライン外に出た場合。GKから再開 |
| コーナーキック | ボールが味方ゴールライン外に出た場合。CKミニゲームへ |

### 9-7. キックオフ・得点後の再開

| 状況 | 処理 |
|---|---|
| 試合開始 | 初期フォーメーション配置。先攻チームのセンターHEXのコマがボールを保持 |
| 得点後 | 両チーム初期フォーメーションに戻る。失点チームのキックオフで再開 |
| 後半開始 | 両チーム初期フォーメーションに戻る。後攻チームのキックオフで再開 |

### 9-8. カード・退場

実装しない。ファウルはFK/PKのトリガーとしてのみ機能する。

### 9-9. ハーフタイム

| 項目 | 内容 |
|---|---|
| 演出内容 | 前半スタッツ表示（支配率、シュート数、パス成功率等） |
| フォーメーション変更 | 不可 |
| 交代 | 不可（後半のターン中に実行） |

### 9-10. 勝利条件
- ボールをゴールに入れる＝得点
- 90ターン終了時に得点の多い方が勝利
- 同点：延長（前後半各5ターン＝計10ターン）→ PKミニゲーム

## 10. ミニゲーム仕様

本編と同じ**同時入力・HEX基準**の設計とし、操作感のギャップをなくす。

### 10-1. フリーキック（FK）ミニゲーム

| 項目 | 内容 |
|---|---|
| 発生条件 | アタッキングサードでのファウル（PA外） |
| 方式 | **同時選択型**（本編の同時入力と統一） |

**攻撃側の入力：**
1. ゴールを6ゾーンに分割（左上・左下・中央上・中央下・右上・右下）
2. キッカーが狙うゾーンを1つ選択
3. 「直接狙う」か「壁を越える（ロブ）」かを選択

**守備側の入力：**
1. 壁の高さを選択（低い＝左右に広い / 高い＝正面を厚くカバー）
2. GKがダイブするゾーンを6択から選択

**判定：**
- 壁が「低い」→ ロブは通るが左右は壁でブロック確率UP
- 壁が「高い」→ 正面がほぼ塞がるがサイドが空く
- キッカーのコストとGKのコストで基本式 (x−y+3)×Ω（Ω=19）を適用
- GKが同ゾーンを選んだ場合：セービング成功
- GKが隣接ゾーンを選んだ場合：GKコスト3なら50%セーブ、コスト2なら25%、コスト1なら不可
- GKが非隣接ゾーン：セーブ不可

### 10-2. コーナーキック（CK）ミニゲーム

| 項目 | 内容 |
|---|---|
| 発生条件 | ボールがゴールライン外に出た（守備側蹴り出し）、またはGKキャッチ失敗 |
| 方式 | **同時選択型・マッチアップ読み合い** |

**攻撃側の入力：**
1. PA内に配置する攻撃コマを3枚選択（フィールドの11枚から）
2. クロスの落下ゾーンを選択（ニア・中央・ファー）
3. 3枚それぞれに「飛び込むゾーン」を指示（ニア・中央・ファーのいずれか）

**守備側の入力：**
1. PA内に配置する守備コマを3枚選択（フィールドの11枚から）
2. 3枚それぞれに「守るゾーン」を指示（ニア・中央・ファーのいずれか）

**判定：**
- 落下ゾーンに飛び込んだ攻撃コマと、同ゾーンを守った守備コマの**1対1マッチアップ**を解決
- 攻撃コマのコスト vs 守備コマのコストで基本判定式（Ω=12）を適用
- 攻撃側が勝った場合：ヘディングシュート → ゴールに向けてシュート成功チェック（§7-2④）を簡易実行
- 守備側が勝った場合：クリア成功。ボールはPA外に出て相手ボールで再開
- 同ゾーンに守備コマがいない場合：フリーヘッダー → シュート成功チェックにボーナス+20
- 攻撃コマが落下ゾーンにいない場合：ボールは流れてGKがキャッチ

> **設計意図**：「誰を3枚選ぶか」「どのゾーンに配置するか」の二重の読み合いが発生する。コスト3のDFでマークするか、ゾーンを分散するか。攻撃側も高コストFWで正面突破するか、WGをファーに走らせてミスマッチを狙うか。

### 10-3. PKミニゲーム

#### A. 試合中PK（PA内ファウル起因）

| 項目 | 内容 |
|---|---|
| キッカー操作 | ゴール6ゾーンからコースを選択 |
| GK操作 | 同じ6ゾーンからダイブ方向を選択 |
| 判定式 | x=キッカーコスト、y=GKコスト、Ω=19 |
| 成功時 | 得点→キックオフで再開（§9-7） |
| 失敗時 | 50%でCK、50%でゴールキック |

> PK失敗後にこぼれ球からの試合続行ではなく、CKまたはGKで仕切り直す。試合中PKは盤面リセットを伴うため、PK判定前後でコマ配置が乱れない設計を優先。

#### B. PK戦（延長後の同点決着）

| 項目 | 内容 |
|---|---|
| 形式 | 各チーム5本ずつ交互。同点ならサドンデス |
| キッカー選択 | フィールド＋ベンチから蹴る順を5人指定 |
| 操作・判定 | 試合中PKと同じ（6ゾーン同時選択、Ω=19） |
| GK隣接セーブ | GKコスト3→隣接ゾーン50%、コスト2→25%、コスト1→不可 |

### 10-4. ミニゲーム共通ルール
- ミニゲーム中は本編のターン消費なし
- 結果は即座にメインボードに反映
- 時代スキルはミニゲーム中は使用不可
